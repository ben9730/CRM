**Migration 1: "create_private_schema_and_foundation"**

Apply a single migration with:

1. Create `private` schema (for security definer functions, not exposed via PostgREST):
   ```sql
   CREATE SCHEMA IF NOT EXISTS private;
   ```

2. Create `accounts` table (team orgs that own CRM data):
   - `id` uuid PK DEFAULT gen_random_uuid()
   - `name` text NOT NULL
   - `created_at` timestamptz NOT NULL DEFAULT now()
   - `updated_at` timestamptz NOT NULL DEFAULT now()

3. Create `account_members` table (users belong to accounts):
   - `id` uuid PK DEFAULT gen_random_uuid()
   - `account_id` uuid NOT NULL FK -> accounts(id) ON DELETE CASCADE
   - `user_id` uuid NOT NULL FK -> auth.users(id) ON DELETE CASCADE
   - `role` text NOT NULL DEFAULT 'member' CHECK (role IN ('admin', 'member'))
   - `created_at` timestamptz NOT NULL DEFAULT now()
   - UNIQUE(account_id, user_id)
   - Indexes: account_members_user_idx(user_id), account_members_account_idx(account_id)

4. Create `profiles` table (extends auth.users with CRM-specific data):
   - `id` uuid PK FK -> auth.users(id) ON DELETE CASCADE
   - `full_name` text
   - `avatar_url` text
   - `created_at` timestamptz NOT NULL DEFAULT now()
   - `updated_at` timestamptz NOT NULL DEFAULT now()

5. Create the security definer function `private.is_account_member(p_account_id uuid)`:
   - RETURNS boolean
   - LANGUAGE sql
   - SECURITY DEFINER
   - SET search_path = ''
   - Body: SELECT EXISTS (SELECT 1 FROM public.account_members WHERE account_id = p_account_id AND user_id = (SELECT auth.uid()))
   - This function bypasses RLS on account_members for the membership check — critical for performance (7ms vs 11,000ms per the research).

6. Create the `handle_new_user()` trigger function:
   - SECURITY DEFINER, SET search_path = ''
   - On INSERT into auth.users:
     a. INSERT into profiles(id, full_name) using NEW.id and NEW.raw_user_meta_data->>'full_name'
     b. Find the first account (SELECT id FROM public.accounts LIMIT 1)
     c. If found, INSERT into account_members(account_id, user_id, role) with 'member' role, ON CONFLICT DO NOTHING
   - This auto-joins new users to the demo account so RLS returns data immediately.

7. Create trigger `on_auth_user_created` AFTER INSERT ON auth.users FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user()

8. Enable RLS on accounts, account_members, and profiles:
   - accounts: SELECT for authenticated where private.is_account_member(id)
   - account_members: SELECT for authenticated where user_id = (SELECT auth.uid()) OR private.is_account_member(account_id)
   - profiles: SELECT for all authenticated (true), UPDATE only own profile ((SELECT auth.uid()) = id)
   - All tables: deny anon access (FOR ALL TO anon USING (false))

Follow the exact SQL from 02-RESEARCH.md "Complete Schema" and "RLS Policy Template" sections. Use `(SELECT auth.uid())` wrapper everywhere — never bare `auth.uid()`.

**Migration 2: "create_crm_entity_tables"**

Apply a migration with all CRM entity tables. Follow the exact schema from 02-RESEARCH.md "Complete Schema" section:

1. `pipeline_stages` table:
   - id uuid PK, account_id uuid FK, name text, display_order int, probability int (0-100 CHECK), is_won boolean, is_lost boolean, color text, created_at timestamptz
   - Index: pipeline_stages_account_idx(account_id, display_order)
   - NO soft delete (hard delete, protected by FK constraints from deals)

2. `organizations` table:
   - id uuid PK, account_id uuid FK, name text NOT NULL, type text CHECK IN ('hospital','clinic','lab','other'), website, phone, address, city, state, tags text[], notes, deleted_at timestamptz, created_at, updated_at, created_by uuid FK, updated_by uuid FK
   - tsvector: search_vector GENERATED ALWAYS AS (to_tsvector('english', coalesce(name,'') || ' ' || coalesce(city,'') || ' ' || coalesce(notes,''))) STORED
   - Indexes: organizations_account_idx, organizations_search_idx (GIN), organizations_deleted_idx (partial WHERE deleted_at IS NULL)

3. `contacts` table:
   - id uuid PK, account_id uuid FK, first_name text NOT NULL, last_name text NOT NULL, email, phone, title, tags text[], notes, deleted_at, created_at, updated_at, created_by, updated_by
   - tsvector: search_vector GENERATED ALWAYS AS (to_tsvector('english', coalesce(first_name,'') || ' ' || coalesce(last_name,'') || ' ' || coalesce(email,'') || ' ' || coalesce(title,'') || ' ' || coalesce(notes,''))) STORED
   - Indexes: contacts_account_idx, contacts_email_idx, contacts_name_idx(last_name, first_name), contacts_search_idx (GIN), contacts_deleted_idx (partial)

4. `contact_organizations` junction table:
   - id uuid PK, contact_id uuid FK -> contacts ON DELETE CASCADE, organization_id uuid FK -> organizations ON DELETE CASCADE, role text, is_primary boolean DEFAULT false, started_at date, ended_at date, created_at timestamptz
   - UNIQUE(contact_id, organization_id)
   - Indexes: contact_orgs_contact_idx, contact_orgs_org_idx
   - NO soft delete (hard delete — ephemeral relationship record)

5. `deals` table:
   - id uuid PK, account_id uuid FK, title text NOT NULL, value numeric(12,2), currency text DEFAULT 'USD', stage_id uuid FK -> pipeline_stages, position text DEFAULT 'n', organization_id uuid FK -> organizations ON DELETE SET NULL, owner_id uuid FK -> auth.users, expected_close date, closed_at timestamptz, notes, tags text[], deleted_at, created_at, updated_at, created_by, updated_by
   - Indexes: deals_account_idx, deals_stage_position_idx, deals_org_idx, deals_deleted_idx (partial)

6. `interactions` table:
   - id uuid PK, account_id uuid FK, type text CHECK IN ('call','email','meeting','note'), subject, body, occurred_at timestamptz DEFAULT now(), duration_mins int, contact_id uuid FK, deal_id uuid FK, organization_id uuid FK, user_id uuid FK NOT NULL -> auth.users, created_at
   - CONSTRAINT interaction_linked: at least one of contact_id, deal_id, organization_id must be NOT NULL
   - Indexes: interactions_contact_idx(contact_id, occurred_at DESC), interactions_deal_idx(deal_id, occurred_at DESC), interactions_account_idx(account_id, occurred_at DESC)
   - Soft delete (deleted_at timestamptz)

7. `tasks` table:
   - id uuid PK, account_id uuid FK, title text NOT NULL, description, due_date timestamptz, is_complete boolean DEFAULT false, completed_at timestamptz, priority text CHECK IN ('low','normal','high') DEFAULT 'normal', contact_id uuid FK, deal_id uuid FK, organization_id uuid FK, assignee_id uuid FK, deleted_at, created_at, updated_at, created_by, updated_by
   - Indexes: tasks_due_idx(account_id, due_date) WHERE is_complete = false, tasks_assignee_idx(assignee_id, is_complete), tasks_account_idx
   - Soft delete

**Migration 3: "create_rls_policies_for_crm_tables"**

Apply RLS policies for all CRM entity tables. For each table with soft delete (contacts, organizations, deals, interactions, tasks):

- Enable RLS
- Deny anon: FOR ALL TO anon USING (false)
- SELECT: USING (deleted_at IS NULL AND private.is_account_member(account_id))
- INSERT: WITH CHECK (private.is_account_member(account_id))
- UPDATE: USING (private.is_account_member(account_id)) WITH CHECK (private.is_account_member(account_id)) — NO deleted_at filter on UPDATE to allow soft delete
- DELETE: USING (private.is_account_member(account_id))

For tables without soft delete (pipeline_stages, contact_organizations):
- pipeline_stages: FOR ALL USING (private.is_account_member(account_id)) WITH CHECK (private.is_account_member(account_id))
- contact_organizations: Derive account_id via the contact's account_id. Use a subquery: USING (EXISTS (SELECT 1 FROM contacts c WHERE c.id = contact_id AND private.is_account_member(c.account_id)))
