**1. Create Server Actions file `src/lib/actions/auth.ts`:**

Mark with `'use server'` at the top.

Import `createClient` from `@/lib/supabase/server` and `redirect` from `next/navigation`.

Implement five actions:

a. `signUp(formData: FormData)`:
   - Extract email, password, full_name from formData
   - Call `supabase.auth.signUp({ email, password, options: { data: { full_name } } })`
   - The `data.full_name` is stored in `raw_user_meta_data` and read by the `handle_new_user` trigger
   - If error, return `{ error: error.message }`
   - If success, redirect('/dashboard')
   - No email verification required (per user decision — Confirm Email is OFF)

b. `signIn(formData: FormData)`:
   - Extract email, password from formData
   - Call `supabase.auth.signInWithPassword({ email, password })`
   - If error, return `{ error: error.message }`
   - If success, redirect('/dashboard')
   - Post-login redirect always goes to Dashboard (locked decision)

c. `signOut()`:
   - Call `supabase.auth.signOut()`
   - redirect('/login')

d. `resetPassword(formData: FormData)`:
   - Extract email from formData
   - Call `supabase.auth.resetPasswordForEmail(email, { redirectTo: \`\${process.env.NEXT_PUBLIC_SITE_URL}/auth/callback?next=/update-password\` })`
   - If error, return `{ error: error.message }`
   - If success, return `{ success: 'Check your email for a password reset link.' }`

e. `updatePassword(formData: FormData)`:
   - Extract password from formData
   - Call `supabase.auth.updateUser({ password })`
   - If error, return `{ error: error.message }`
   - If success, redirect('/dashboard')
