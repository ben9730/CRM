  <action>
    Start the dev server (`npm run dev`) and systematically test every screen at each breakpoint using the Playwright MCP browser tools. For each viewport width, take a screenshot and verify no layout breaks exist.

    **Testing Protocol — for each of the 4 prototype screens + contact detail:**

    At **1280px** (desktop):
    - All content visible without horizontal scrollbar (except Kanban intentional scroll)
    - Sidebar expanded by default with full labels
    - Dashboard: 4 metrics in a row, 2/3 + 1/3 split below, full-width activity
    - Contacts table: all columns visible
    - Kanban: all columns visible (may have horizontal scroll for 6 columns — this is acceptable)
    - Contact detail: full-width layout

    At **768px** (tablet):
    - Sidebar should collapse or switch to offcanvas mode
    - Dashboard: metrics may stack to 2x2 grid, pipeline/tasks stack vertically
    - Contacts table: may need horizontal scroll or fewer visible columns
    - Contacts grid: 2 columns instead of 3
    - Kanban: horizontal scroll for columns (column width stays ~280px)
    - Contact detail: single column layout
    - Sheet slide-over: still slides from right but narrower

    At **375px** (mobile):
    - Sidebar is fully hidden (offcanvas mode — hamburger menu opens it)
    - Dashboard: metrics stack vertically (1 column), all widgets stack
    - Contacts table: might need to hide non-essential columns (keep name, org, actions)
    - Contacts grid: 1 column
    - Kanban: horizontal scroll, one column visible at a time
    - Contact detail: fully stacked, compact spacing
    - Sheet: full-width (`w-full`) — no 480px fixed width on mobile
    - Header search: may need to collapse to icon or be smaller

    **Fixes to apply (fix each issue found during testing):**

    Common responsive patterns:
    - Dashboard metrics: `grid grid-cols-1 sm:grid-cols-2 xl:grid-cols-4`
    - Dashboard row 2: `flex flex-col lg:flex-row` for pipeline/tasks
    - Contacts grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
    - Kanban board: `overflow-x-auto` container (already should have this)
    - Sheet: `className="w-full sm:w-[480px]"` (responsive width)
    - Sidebar: shadcn Sidebar automatically handles mobile via `SidebarProvider` (it switches to offcanvas below `md` breakpoint) — verify this works and fix if not
    - Contact detail sections: `grid grid-cols-1 lg:grid-cols-2` for linked deals/tasks side-by-side on desktop, stacked on mobile
    - Tables: wrap in `overflow-x-auto` container on mobile; OR hide non-essential columns using Tailwind `hidden md:table-cell`

    Test each fix after applying. Use Playwright browser_resize to change viewport dimensions.

    IMPORTANT: The goal is "renders correctly without layout breaks" — not pixel-perfect at every size. Acceptable: horizontal scroll on Kanban (intentional), slight column hiding on narrow tables. Unacceptable: content overflow, elements cut off, unreadable text, non-functional interactions.
  </action>
