  <name>Task 2: Build portal chat UI components with markdown rendering</name>
  <files>src/components/portal/PortalChat.tsx, src/components/portal/PortalMessage.tsx</files>
  <action>
Create src/components/portal/PortalMessage.tsx -- bubble message component:
1. 'use client' directive
2. Import Markdown from 'react-markdown' and remarkGfm from 'remark-gfm'
3. Props: `{ role: 'user' | 'assistant'; content: string }`
4. Layout: flex container with `justify-end` for user messages, `justify-start` for assistant messages
5. User bubble: colored background using `bg-primary text-primary-foreground`, rounded-2xl with `rounded-br-sm` (small bottom-right for chat tail effect), max-w-[85%], px-4 py-2.5, text-sm. Render content as plain text `<p>{content}</p>`
6. Assistant bubble: `bg-muted/50 text-foreground`, rounded-2xl with `rounded-bl-sm`, same sizing. Render content through `<Markdown remarkPlugins={[remarkGfm]} components={markdownComponents}>{content}</Markdown>`
7. Define markdownComponents object with Tailwind-styled elements per the RESEARCH.md pattern:
   - p: `mb-2 last:mb-0 leading-relaxed`
   - strong: `font-semibold text-foreground`
   - h1: `mb-2 text-base font-bold`, h2: `mb-1.5 text-sm font-bold`, h3: `mb-1 text-sm font-semibold`
   - ul: `mb-2 ml-4 list-disc space-y-1`, ol: `mb-2 ml-4 list-decimal space-y-1`, li: `leading-relaxed`
   - code: detect block vs inline via `!!className` (react-markdown v10 pattern -- block code has a language-* className). Block: `block rounded-md bg-muted/60 p-3 font-mono text-xs overflow-x-auto`. Inline: `rounded bg-muted/40 px-1 py-0.5 font-mono text-xs`
   - pre: `mb-2 overflow-x-auto rounded-md`
8. Each bubble has `mb-3` spacing between messages

Create src/components/portal/PortalChat.tsx -- main chat client component:
1. 'use client' directive
2. State: messages array (role + content), input string, isLoading boolean, geminiHistory array
3. Container: `h-dvh flex flex-col bg-background` -- dvh shrinks with iOS keyboard, flex-col pushes input to bottom. On desktop, center with `max-w-2xl mx-auto w-full` (Claude's discretion: prevents chat from spanning full width on large screens)
4. Back navigation: A subtle ChevronLeft icon button in the top-left corner, absolute positioned, linking to `/dashboard`. Small, semi-transparent, non-intrusive. Use `lucide-react` ChevronLeft icon, size-5, text-muted-foreground hover:text-foreground. (Claude's discretion per CONTEXT.md)
5. Message area: `flex-1 overflow-y-auto px-4 py-4` -- scrollable, takes remaining space. Auto-scroll to bottom on new messages using a ref + useEffect
6. Empty state: Just the input field visible -- NO greeting text, NO suggestion chips, NO prompts (locked decision from CONTEXT.md). The message area is simply empty.
7. Loading state: When isLoading is true, show a Loader2 spinner (from lucide-react) with `animate-spin` inside the message area, styled like an assistant message position (left-aligned). NOT typing dots (locked decision).
8. Input area: Pinned at bottom via flex layout (NOT position:fixed -- research anti-pattern). Border-t separator. Textarea + send button in a rounded container. Style: `border-t border-border/50 px-3 py-3` with `paddingBottom: 'calc(env(safe-area-inset-bottom, 0px) + 12px)'` for iPhone home bar. Textarea: auto-resize up to 4 lines max, bg-transparent, text-sm. Send button: bg-primary, rounded-lg, disabled when empty or loading.
9. Send message handler:
   - Trim input, skip if empty or loading
   - Add user message to messages array
   - Clear input, set loading true
   - POST to /api/chat with { message, history: geminiHistory }
   - Parse response JSON
   - **Rate limit handling**: If `data.rateLimited` is true, add an assistant message with `data.friendlyMessage` (or fallback "I'm taking a breather -- try again in a minute"). Do NOT throw an error.
   - On success: add assistant message with `data.response`, update geminiHistory from `data.history`
   - On other errors: add assistant message with "Something went wrong. Please try again." (never show raw error strings)
   - Set loading false in finally block
10. Keyboard handling: Enter sends (without Shift), Shift+Enter adds newline
11. Message limit: Keep last 50 messages in the UI array (portal has more screen real estate than the widget's 20)
  </action>
