Refactor `sendMessage` in PortalChat.tsx to accept an optional `textOverride?: string` parameter. When `textOverride` is provided, use it instead of `input` state — this avoids the async setState race condition (CRITICAL: do NOT use `setInput(msg)` then `sendMessage()` — the state won't update in time).

Updated sendMessage signature and key logic:
```tsx
const sendMessage = async (textOverride?: string) => {
  const trimmed = (textOverride ?? input).trim()
  if (!trimmed || isLoading || !sessionId || pendingAction) return

  const userMessage: Message = { role: 'user', content: trimmed }
  setMessages(prev => {
    const updated = [...prev, userMessage]
    return updated.slice(-MAX_MESSAGES)
  })
  // Only clear input when sending from textarea (not quick action)
  if (!textOverride) setInput('')
  setIsLoading(true)
  // ... rest unchanged
}
```

Update the `handleKeyDown` and send button onClick to call `sendMessage()` (no args — reads `input` state as before).

Add a `QUICK_ACTIONS` array constant above the component:
```tsx
const QUICK_ACTIONS = [
```

Render quick action pill chips INSIDE the input area container (the `border-t` div), ABOVE the textarea row, inside the `mx-auto max-w-2xl` wrapper. They should appear as a horizontally-scrollable row of pill buttons — WhatsApp suggested-replies style:
