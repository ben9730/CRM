    - Table: `deal_contacts` with columns: `id uuid PK default gen_random_uuid()`, `deal_id uuid FK deals(id) NOT NULL`, `contact_id uuid FK contacts(id) NOT NULL`, `role text` (e.g., 'primary', 'influencer'), `created_at timestamptz default now()`
    - Unique constraint on `(deal_id, contact_id)` — prevent duplicate links
    - RLS: enable, add policy using `private.is_account_member()` via EXISTS through deals
    - Run via `mcp__plugin_supabase_supabase__apply_migration` with name `add_deal_contacts_junction`
    - After migration, regenerate types: `npx supabase gen types typescript --linked > src/types/database.ts`
    - If type generation fails, manually add the `deal_contacts` type to `src/types/database.ts`

    **2. Update `src/lib/types/app.ts` — add deal types:**
    - `DealRow` = `Tables<'deals'>`
    - `PipelineStageRow` = `Tables<'pipeline_stages'>`
    - `DealWithRelations` = deal row + `{ stage: PipelineStageRow; organization?: { id: string; name: string }; contacts?: { id: string; first_name: string; last_name: string }[] }`

    **3. Create `src/lib/queries/pipeline-stages.ts`:**
    - `getPipelineStages()` — returns all stages ordered by `display_order`
    - Query: `supabase.from('pipeline_stages').select('*').order('display_order')`
    - No soft-delete filter needed (pipeline_stages has no `deleted_at` column)

    **4. Create `src/lib/queries/deals.ts`:**
    - `getDeals()` — returns all active deals with stage and org joins
    - Query: `supabase.from('deals').select('*, pipeline_stages(id, name, color, display_order, is_won, is_lost), organizations(id, name)').is('deleted_at', null)`
    - `getDeal(id: string)` — single deal with full relations including contacts via `deal_contacts`
    - `getDealsByOrganization(orgId: string)` — deals for a specific org
    - `getDealsByContact(contactId: string)` — deals linked via `deal_contacts`

    **5. Create `src/lib/actions/deals.ts`:**
    - `'use server'` directive
    - Zod schema: `title` required, `stage_id` UUID required, `value` number optional, `expected_close` date string optional, `organization_id` UUID optional, `contact_ids` string[] optional, `notes` optional
    - `createDeal(_prevState: ActionState, formData: FormData)`:
      1. Validate with Zod
      2. `getAccountId` for `account_id`
      3. Insert deal row with `account_id`, `created_by`, `updated_by`, `owner_id` = current user
      4. For each `contact_id`, insert into `deal_contacts`
      5. `revalidatePath('/deals')`
    - `updateDeal(_prevState: ActionState, formData: FormData)` — update deal row + re-sync deal_contacts
    - `deleteDeal(id: string)` — soft-delete, `revalidatePath`
    - `moveDealStage(dealId: string, newStageId: string)` — lightweight action for Kanban drag:
      1. Auth check
      2. `supabase.from('deals').update({ stage_id: newStageId, updated_by: user.id }).eq('id', dealId)`
      3. `revalidatePath('/deals')`
      4. Return `{ error?: string }` for rollback signal
