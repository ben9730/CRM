<objective>
Build complete CRUD for organizations and contacts wired to the live Supabase backend. This plan creates the shared foundation (account helper, app types, shared UI components) that all subsequent plans depend on, then implements full list/create/edit/delete/search/filter for both entities. Organizations get a detail page showing linked contacts and deals. Contacts support multi-org junction, free-form tags, full-text search, and classic pagination.

Purpose: Organizations and contacts are the foundational CRM entities — every deal, interaction, and task links back to them. This plan replaces mock data with live Supabase queries and establishes the Server Component + Server Action patterns that plans 03-02 and 03-03 will follow.

Output: Working organizations and contacts CRUD with search, filter, tags, pagination, and detail pages — all wired to Supabase.
</objective>
